# AGENTS.md — Darles Games Interactive Web Products

## Роль Codex

Ты работаешь с публичным репозиторием-библиотекой демонстрационных интерактивных веб-продуктов Darles Games.

Главная задача репозитория — показывать отдельные демо по постоянным URL и давать главному сайту Darles Games прямые ссылки на каждое демо.

## Основной принцип

Репозиторий является каталогом автономных HTML5-продуктов.

Каждый продукт:

- находится в собственной папке в корне репозитория;
- имеет собственный `index.html`;
- открывается по отдельному URL GitHub Pages;
- не зависит от файлов другого продукта;
- не должен ломаться при добавлении или обновлении соседних продуктов.

Новые демо добавляются обычным копированием новой папки продукта в корень локального репозитория.

## Запрещено без прямого запроса пользователя

- переименовывать папки продуктов;
- переносить продукты в другую структуру;
- объединять исходники разных продуктов;
- переписывать механику или дизайн демо;
- заменять технологии продукта;
- добавлять npm, Vite, React, сборщик или серверную часть только ради каталога;
- добавлять открытую лицензию MIT, Apache, GPL и аналогичные;
- публиковать ключи, токены, персональные данные или клиентские материалы;
- удалять существующие файлы продукта как «лишние» без проверки;
- менять продукт только ради единообразия каталога.

Допустимы минимальные исправления путей и запуска, если продукт не работает внутри GitHub Pages. Каждое такое изменение нужно перечислить в итоговом отчёте.

## Рекомендуемая структура корня

```text
/
├── index.html                 # Главная страница каталога
├── README.md
├── AGENTS.md
├── .nojekyll
├── catalog/
│   ├── catalog.css
│   ├── catalog.js
│   ├── products.json
│   └── assets/
├── 00_interactive-web-solution-quiz-v1.0/
├── 00_interactive-web-solution-quiz-v2.0/
├── 02_standart_quiz__diagnostic/
├── 03_magic_quiz_diagnostic/
├── 04_catch_discount/
└── creeting_cards/
```

Папки продуктов остаются в корне, потому что пользователь добавляет новые демо простым копированием папки в локальный каталог репозитория.

## Каталог

Главная страница должна:

- читать данные из `catalog/products.json`;
- группировать продукты по категориям;
- показывать название, краткое описание, статус и кнопку открытия;
- вести по кнопке на `./<имя-папки>/`;
- открывать демо в новой вкладке, чтобы пользователь не терял каталог;
- корректно работать на мобильных и настольных устройствах;
- не требовать сборки;
- не зависеть от внешних фреймворков и CDN без необходимости.

Первый релиз каталога — на русском языке. Структура данных и интерфейса не должна мешать последующему добавлению английской локализации.

## Метаданные продукта

Каждая запись в `catalog/products.json` должна содержать минимум:

```json
{
  "id": "unique-product-id",
  "title": "Название для каталога",
  "folder": "folder-name",
  "category": "category-id",
  "categoryTitle": "Название категории",
  "description": "Короткое понятное описание",
  "status": "Демо",
  "order": 10,
  "featured": false
}
```

`folder` должен точно совпадать с реальным именем папки, включая регистр, символы, точки и возможные опечатки.

## Добавление нового продукта

Когда в корень скопирована новая папка:

1. Проверить наличие `index.html`.
2. Проверить запуск через локальный HTTP-сервер.
3. Не менять имя папки без прямого разрешения.
4. Добавить продукт в `catalog/products.json`.
5. Выбрать существующую категорию или добавить одну новую только при необходимости.
6. Создать карточку через данные каталога, а не вручную в HTML.
7. Проверить прямой URL продукта.
8. Проверить, что добавление не повлияло на остальные демо.
9. Обновить README только при изменении структуры или состава категорий.

Если папка не содержит рабочий `index.html`, не пытайся полностью переделать продукт. Зафиксируй проблему и предложи минимальное исправление.

## Локальный запуск

Не открывать каталог и продукты только через `file://`, если они используют `fetch`, ES modules или другие функции, требующие HTTP.

Основная команда для Windows:

```powershell
py -m http.server 8080
```

Запускать её из корня репозитория, затем открывать:

```text
http://localhost:8080/
```

Допустимые альтернативы:

```powershell
python -m http.server 8080
npx serve .
```

Команда `py -m local.server: 8080` неверна и не должна использоваться.

## GitHub Pages

Каталог должен работать как статический сайт GitHub Pages без отдельного backend.

Базовый вариант публикации:

- ветка: `main`;
- источник Pages: корень ветки;
- основной URL: `https://darlesgames.github.io/<repository-name>/`;
- URL продукта: `https://darlesgames.github.io/<repository-name>/<folder-name>/`.

Все внутренние ссылки и ресурсы должны использовать относительные пути. Нельзя предполагать, что сайт опубликован в корне домена.

Проверять особенно внимательно:

- пути вида `/assets/...`;
- регистр имён файлов;
- `fetch('/file.json')`;
- импорт модулей от корня домена;
- ссылки с жёстко заданным localhost;
- пробелы и специальные символы в путях.

Предпочтительные пути:

```text
./assets/image.webp
../shared/file.js
./catalog/products.json
```

## Автономность продуктов

Общие файлы каталога не должны быть обязательными для запуска демо.

Не подключай общий `shared`-код внутрь продукта, если это не требуется пользователем. Иначе изменение каталога может сломать несколько демо одновременно.

Каталог может использовать собственные общие стили и компоненты только внутри `catalog/`.

## Обложки

Если у продукта нет обложки:

- сначала использовать существующий скриншот или промо-изображение продукта;
- если подходящего изображения нет, временно использовать аккуратную текстовую заглушку карточки;
- не скачивать случайные изображения из интернета;
- не добавлять чужие материалы с неясной лицензией.

## Проверка перед завершением задачи

Обязательно проверить:

- корневой каталог открывается через локальный HTTP-сервер;
- все карточки ведут в существующие папки;
- все шесть текущих продуктов открываются напрямую;
- отсутствуют ошибки 404 по основным ресурсам;
- каталог работает с базовым URL репозитория, а не только на localhost;
- карточки читаются на ширине 360 px;
- клавиатурный фокус видим;
- ссылки и кнопки имеют понятные названия;
- в репозитории нет секретов и локальных абсолютных путей;
- продукты не были переименованы или существенно изменены без разрешения.
